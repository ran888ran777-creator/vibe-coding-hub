## 🧭 Коротко

В репозитории [`serejaris/ris-claude-code`](https://github.com/serejaris/ris-claude-code) есть полезная рамка: *Personal Corp*. Идея простая — соло-разработчик может использовать GitHub как базовую операционную систему, а AI-агентов как исполнителей отдельных ролей и циклов.

## 🎯 Почему это полезно

Обычный режим vibe coding часто разваливается на хаотичные чаты и несвязанные задачи. Personal Corp полезен тем, что возвращает операционный контур:

- где живут задачи;
- как они маршрутизируются;
- как планируется неделя;
- как собирается ретро;
- как решения возвращаются в систему.

## ⚡ Базовый цикл

По публичному README `ris-claude-code` фреймворк строится вокруг таких блоков:

| Блок | Смысл |
| --- | --- |
| `project-init` | Зафиксировать стартовые правила и конфигурацию |
| `task-routing` | Понять, куда должна попасть задача |
| `weekly-planning` | Превратить backlog и выводы ретро в приоритеты |
| `weekly-retro` | Не терять уроки и наблюдения после выполнения |

Это ценно даже без прямого копирования скиллов: сама последовательность уже улучшает управление проектом.

## 🔧 Как адаптировать под личный репозиторий

Для личного хаба достаточно минимальной версии:

1. `README.md` отвечает за внешнюю рамку проекта.
2. `CLAUDE.md` задаёт поведение ассистента.
3. GitHub Issues хранят задачи.
4. Один weekly review файл или issue фиксирует выводы недели.

Если это работает, можно добавлять автоматизацию дальше.

## 💡 Что особенно ценно в подходе serejaris

Помимо репозитория со скиллами, у [`serejaris/sereja.tech`](https://github.com/serejaris/sereja.tech) видно устойчивое направление тем:

- многоагентные системы;
- GitHub как контур управления;
- автоматизация аналитики и расписаний;
- data layer для агентов;
- агентно-читаемые форматы вроде `llms.txt`.

Это хороший ориентир, куда расширять хаб дальше.

## ⚠️ Ограничения

- Personal Corp не нужен, если у вас пока нет повторяемой нагрузки.
- Слишком ранняя автоматизация может усложнить простой проект.
- Если правила и артефакты не поддерживать, система быстро превращается в формальность.
