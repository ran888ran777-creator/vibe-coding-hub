## 🧭 Коротко

Репозиторий [`serejaris/justdoit`](https://github.com/serejaris/justdoit) полезен не только как Codex skill, но и как сама идея: перед исполнением агент должен собрать execution pack, а не сразу бежать править код.

## 🎯 В чём суть паттерна

По README `justdoit` сначала:

- просматривает проект;
- готовит артефакты планирования;
- проверяет зависимости между шагами;
- и только после этого предлагает запуск.

Это сильный паттерн для задач формата «сделай что-нибудь с этим репо».

## ⚡ Какие артефакты нужны

В `justdoit` напрямую названы три файла:

| Артефакт | Зачем нужен |
| --- | --- |
| `plans.md` | Зафиксировать этапы и зависимости |
| `status.md` | Отражать текущее состояние задачи |
| `test-plan.md` | Отдельно думать о проверках, а не вспоминать о них в конце |

Даже если вы не используете именно этот skill, сама структура полезна почти в любом долгом run.

## 🔧 Как использовать у себя

Для задач средней и высокой неопределённости применяйте такой цикл:

1. Сначала обзор репозитория и ограничений.
2. Потом компактный execution pack.
3. Затем подтверждение подхода.
4. И только потом изменения.

Пример хорошего входа:

```text
Сначала изучи проект и подготовь execution pack:
- plans.md
- status.md
- test-plan.md
Ничего не меняй, пока не покажешь план.
```

## 💡 Когда это особенно полезно

- Непонятно, в каких файлах лежит логика.
- В задаче есть риск затронуть CI, деплой или данные.
- Нужно делегировать работу другому агенту.
- Вы хотите сохранить ход мысли не только в чате.

## ⚠️ Когда не нужно усложнять

Execution pack не нужен для совсем маленьких правок, вроде:

- поменять один заголовок;
- исправить опечатку;
- обновить один URL;
- добавить один CSS-класс без побочных эффектов.

Там он только замедлит работу.
